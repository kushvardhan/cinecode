TOP-LEVEL PRINCIPLES (Must be enforced)
---------------------------------------
- Keep homepage minimal: small headline, single line subline, 2 CTAs, cinematic hero.
- Each section MUST feel like a film-beat: introduction → reveal → payoff.
- Respect `prefers-reduced-motion` globally.
- Lazy-load heavy libs (three, gsap) using dynamic imports in client components ONLY.
- Keep initial (critical) JS low. Target: **≤ 250 KB** preload JS for first paint.
- Provide fallback poster images or HeroLite for low-end devices.
- Add a “Motion Intensity” control (Low/Medium/High/Off) in user settings.
- Accessibility: keyboard-first navigation, visible focus, contrast, alt tags.

FONTS & TYPOGRAPHY (hero-worthy)
--------------------------------
You said you like Sheriyans hero font. Exact font may be commercial. Recommended approach:

Primary Display (hero & headings)
- Preferred (commercial): **Neue Montreal** (if you license it)
- Free alternatives:
  - **Space Grotesk** (Google Fonts - nice cinematic feel)
  - **Poppins** / **Montserrat Alternates** (clean, geometric)
  - **Satoshi** (if available and licensed) — use as a display if you can obtain.

Body (readable)
- **Inter** (variable, performant, readable)
- or **Roboto / System UI** fallback

Font usage:
- Use variable fonts to reduce requests (e.g., Inter Variable)
- Preload hero font with `rel="preload"` in head for LCP
- Provide `font-display: swap` as fallback
